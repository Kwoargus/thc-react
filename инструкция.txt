Для создания БД в PostgreSQL (уже должен быть установлен) надо
1. открыть DBeaver и подключиться к PostgreSQL
2. создать схему task_hardness_calculator
3. из меню "Редактор SQL" открыть "новый Редактор SQL"
4. из файла THC_dump.sql всё содеожимое копипастом перенести в открывшееся окно нового Редактора SQL
5. в окне Редактора SQL нажать кнопку с изображением листка и треугольной кнопка на нём - "выполнить SQL скрипт (Alt+X)"
6. если всё сделано правильно, то в окне консоли (в нижней части окна DBeaber) побегут строки сообщений о выполнении команд,
   а по окончании в схеме task_hardness_calculator в разделе "таблицы" появятся 4 новых таблицы с данными.

Для запуска бэкенда необходимо:
1. Установить JDK, лучше версию 11-coretto и задать значения переменных окружения JAVA_HOME и
2. Java архив переименовать в taskhardnesscalculator-0.0.1-SNAPSHOT.rar (rar архив) открыть архив как папку,
   прейти в файле src/resources/application.yml и datasource вписать ваши локальные username и password; в разделе server параметрe port нужно задать значение 8081
3. В каталоге (лучше создать отдельный) где лежит java-архив taskhardnesscalculator-0.0.1-SNAPSHOT.jar открыть терминал
4. Выполнить команду
    java -jar taskhardnesscalculator-0.0.1-SNAPSHOT.jar

а последней строкой вывода будет сообщение о том приложение TaskHardnessCalcApplication стартовало:
2024-01-11 11:45:19.992  INFO 8736 --- [           main] r.a.t.TaskHardnessCalcApplication        : Started TaskHardnessCalcApplication in 10.585 seconds (JVM running for 11.141)

после этого приложение должно отвечать на вызовы фронта - в консоли фронта сообщение о недоступности (коды 400 или 500) сервера должно поменяться на 200

если все настройки выполнены правильно, то при запуск бэка и фронта, при выполнении сохранения значений факторов названия и описания задачи и пр.
в БД, в соответствующей таблице должна появляться новая строка с введёнными на фронте значениями полей.
